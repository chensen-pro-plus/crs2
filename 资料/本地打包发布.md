如果你想跳过自动化流程，手动在本地编译并替换服务器文件，请按以下步骤操作：

1. 编译前端（生成静态资源）
这是最关键的一步，你需要生成最新的 dist 目录。

bash
# 1. 进入前端目录
cd web/admin-spa
# 2. 安装依赖（如果没装过）
npm install
# 3. 执行编译
npm run build
编译成功后，你会看到目录 web/admin-spa/dist/，里面包含了 index.html 和 assets 文件夹。

2. 替换服务器文件
这一步取决于你的服务器部署方式。通常你需要做的是：

情况 A：你只想更新前端界面
将本地刚才生成的 web/admin-spa/dist/ 文件夹里面的所有内容，上传并覆盖服务器上对应的目录。

源路径：你的本地项目/claude-relay-service2/web/admin-spa/dist/
目标路径：取决于你的服务器配置，通常是项目目录下的 web/admin-spa/dist/。
提示：上传后通常不需要重启后端服务，刷新浏览器即可看到新界面（因为静态文件是热读取的，除非你配合 Nginx 做了强缓存）。

情况 B：你修改了后端代码 (src/ 或根目录)
如果你不仅改了前端，还改了后端 Node.js 代码：

上传代码：将本地的 src/ 文件夹（以及根目录的 config/, 
package.json
 等）上传覆盖服务器对应文件。
安装依赖：如果 
package.json
 有变动，在服务器项目根目录执行 npm install。
重启服务：必须重启 Node.js 服务才能生效。
例如：pm2 restart claude-relay-service 或 systemctl restart claude-relay。
